Jason Zhou 分享了他使用 Codebase Memory MCP 工具的实际体验——这个工具用纯 C + tree-sitter 构建代码图索引，无需任何模型参与，索引速度极快（Linux 内核 28M 行约 3 分钟，普通仓库数秒）。通过为 Agent 提供代码结构地图（get_architecture、search_graph、trace_path、query_graph 等工具）和一个精巧的 grep hook 设计，在部分任务中降低了约 50% 的 token 消耗。作者认为嵌入（embedding）方案不实用（语义检索定位不准、无法跨文件追踪调用链），而 grep + read 模式在大代码库中会浪费大量 token 打开无关文件。Codebase Memory 的方案是在 Agent 的 grep 工具上挂一个 pre-tool-use hook——当 Agent grep 某个函数时，grep 正常执行，同时 hook 在图索引中查找该查询并折叠结构信息到搜索结果中。Agent 不需要记住调用特殊 MCP 工具，它 grep 就行了，图自动跟上。
