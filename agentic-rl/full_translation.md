# Agentic RL: 框架与最佳实践

翻译校准版：密度与 article.md 匹配，保留全部原文信息但去掉过度逐句翻译。

---

Agentic RL 是当前 AI 研究热点。LLM 从单轮任务走向多轮自主交互，需要在长时间跨度上推理、调用工具、与用户交互、适应反馈。本综述研究 RL 训练 LLM Agent 的最新进展，识别实用设计原则。

**Agent 组件**

Agent = 在 Agent 循环中运行的 LLM。系统包含 LLM 骨干网络、指令、工具、环境，在循环中生成输出、执行工具调用、消化反馈、重复，直到满足终止条件。这些组件构成 Agent 框架。

LLM 骨干网络需要高级指令遵循、工具调用和推理能力，推理模型（产生思考轨迹）尤其有用。Agent 必须分解复杂问题、自我反思和恢复。

指令提供问题信息和解决策略（分解步骤、维护待办列表等）。平衡简洁和特异性。

工具与环境是 Agent 对外接口。工具调用可用特殊 token 表示。例如 Qwen3 用 XML 分隔符封装工具定义、调用和响应。工具确定地追加 token 或非确定地更新环境状态。

Agent 循环每步先生成 token，工具调用时暂停→执行→追加结果→继续。步后检查终止条件（最大步数、测试、LLM 输出的完成信号）。

额外组件：上下文管理（长期任务压缩或截断），记忆系统（持久化上下文跨任务会话）。

**RL 训练**

RL 训练交替 rollout（采样多个补全+计算奖励）和策略更新（用 rollout 和损失函数更新权重）。优化器包括 GRPO、PPO、REINFORCE。长周期任务中 PPO 重新受青睐——超长轨迹被压缩分割后 GRPO 不如 PPO 稳定。

Agentic rollout 多轮交互输出，每一步通过工具调用与环境交互。采样是基础设施难题——rollout 长度方差高，某些因交互轮多或环境慢而耗时更长。训练用异步 RL 和分离式架构。

**MDP 公式**

单轮 RL：状态=token 上下文，动作=下一个 token，转移函数确定追加 token，奖励=终局结果奖励。

多轮 Agentic RL：状态=LLM 可见上下文+环境外部状态。动作=文本或工具调用序列。转移函数可能非确定。奖励=终局+中间步骤奖励。

环境隔离关键：每个 rollout 需要独立环境实例（Docker/沙箱），避免轨迹间干扰。扩展环境是系统挑战——512 个并发 Docker 会撑爆 API 服务器。Kubernetes 调度是常见方案。

**ToRL：工具集成推理 RL**

ToRL 使用 RL-Zero 设置，从预训练基座模型开始，加代码解释器工具，让模型通过奖励探索学会写代码推理。提示词引导用 NL+代码块融合解答。

生成时 ```python 包裹代码→```output 标签触发暂停→执行→追加结果。每问题限 C 次调用（大多 C=1）。代码执行错误返回仅最后一行，代码输出从损失屏蔽，代码在沙箱执行。

奖励：正确+1，错误-1，不可执行代码-0.5。惩罚不可执行代码不益于性能——可能让模型过于保守。纯结果奖励最优。

在 29K 数学题上用 GRPO 训练。LIMR 筛选可学样本。Qwen2.5-Math-7B 上 14.7% 绝对提升。训练中代码解题比例从 40%→80%，成功执行率也提升。模型学会从错误消息反思、用代码+NL 验证。

**AgentGym-RL**

开源框架，基于结果的 RL，模块化。三组件：环境、Agent、训练。环境覆盖多个领域，每个以独立服务运行暴露统一 HTTP 接口，即插即用。

工程优化：WebArena 单浏览器→子进程架构（单服务器多 Chromium）；SciWorld 修改初始化支持并行；WebArena 全重置接口；verl 扩展支持多轮轨迹。

**ScalingInter-RL**

课程学习：训练分 N 阶段（δ=80 步/阶段），交互预算 8→12→15 轮。达到上限终止交互让 Agent 给出解。目标最大化累积奖励但约束放松。

7B 模型搜索 26% 和深度研究 38.25% 成功率，超 GPT-4o 和 70B 模型。RL 在结构化任务（TextCraft、SciWorld）最有效。早期过度探索有害——短轮数限制提供稳定信号。递增预算迁移已学模式。

固定大预算（10 轮）初始好但崩溃。RL 训练 Agent 有更强测试时扩展能力。优化器选择重要——3B+GRPO 优于 7B+REINFORCE。

**Agent-R1**

结构化轨迹格式。步级块存储：观察→推理→动作→工具调用详情→奖励。先前步数据静态不参与计算图，精确 token 恢复避免 token 化漂移。

Agent-R1-7B 是首个通过 RL 训练的 7B 模型，多基准超 GPT-4o。ALFWorld+62.4%、DB+43.2%、KG+14.6%、OS+31.1%、WebShop+62.4%。跨领域泛化——不同评判模型和提示词保持性能。

**AgentRL**

多任务多轮异步 RL，跨环境（文本、具身、网页）训练，任务 5-25 轮。二元奖励+步骤效率奖励。

三设计：跨策略采样（KL 门控丢弃离策略轨迹）、任务级优势归一化（全局归一化避免困难任务主导）、陈旧控制（重放缓冲+KL 门控）。GRPO+全部组件最优。

SWE-Bench 15.5%→33.5%，GAIA 0%→20%。未见过验证集上表现良好——Agent 学到可泛化技能。

**AutoForge**

从工具文档（OpenAPI/Markdown）自动生成合成环境。两步：环境构造→模拟用户 Agent 生成样本。训练：SFT 冷启动→RL 优化。奖励：最终状态匹配+1，否则 0。

ERPO（环境相对策略优化）三改动：交织思考保留推理轨迹；屏蔽错误模拟用户行为；环境级优势估计（跨环境汇集标准差提高异常值鲁棒性）。

使用 Qwen3-235B 合成 10 环境 1078 任务。微调 Qwen3-30B 域内超更大模型接近 Gemini-2.5-Pro。域外泛化（不同格式/工具集/跨语言英中）仍有提升。

**RAGEN & StarPO：回声陷阱**

StarPO 将整个多轮轨迹（含推理轨迹+动作+反馈）作为优化单元。保留中间推理轨迹但不屏蔽环境 token。

回声陷阱：Agent 过拟合自生成推理模式，行为崩溃。表现：奖励停滞、组内奖励变异性和熵下降、梯度范数飙升。

StarPO-S 稳定变体：删除 KL 正则化、Clip-Higher 防熵崩溃、高方差任务选择（奖励标准差排序留高信号）。

**RAGEN-2：模板崩溃**

模板崩溃比回声陷阱更隐蔽。Agent 对每个输入保持高熵但使用与输入无关的固定模板。单纯熵指标不够——需要互信息测跨输入推理深度。

动态 SNR 过滤：每任务多轨迹→计算组方差→按方差保留高信号比例（p=0.9）。移除低信号提示词贡献，避免模板崩溃。

**6 条设计原则**

模块化接口（HTTP/ToolEnv/函数调用）、轨迹结构（步级而非扁平）、动作掩码（Agent token 用 RL 目标+环境 token 用 SFT 目标）、过程奖励谨慎使用、优势跨环境归一化而非组内、稳定性保障（高方差任务筛选/课程学习/跨策略采样应对回声陷阱和模板崩溃）。
