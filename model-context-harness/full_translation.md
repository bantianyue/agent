# Externalization in LLM Agents: 论文核心框架

## 核心论点

LLM Agent 的进化轨迹是从"内部权重"经"上下文窗口"到"外部基础设施"的逐步外化（externalization）。这不是一个关于工程便利性的论点，而是一个关于**可靠智能从何而来**的论点：不是来自越来越大的模型，而是来自对任务需求的系统性重构——将认知负担从模型的内部计算逐步迁移到持久、可检查、可重用的外部结构中。

论文以 Norman 的"认知工具（cognitive artifacts）"理论为锚点：购物清单不是扩大了生物记忆容量，而是把困难的回忆问题变成了识别问题。同样，Agent 基础设施不是在"加强"模型，而是在**重构问题表征**。

## 历史弧线：Weights → Context → Harness

### 第一层：能力在权重中（~2022-2023）
能力几乎完全等同于模型参数。预训练将统计规律、世界知识、推理习惯压缩进权重。微调和偏好优化进一步塑造模型行为。优点是快速推理、紧凑部署、强泛化。缺点：更新事实需要重训练、审计困难、个性化尴尬。

### 第二层：能力在上下文中（~2023-2024）
提示工程、few-shot、CoT、ReAct 等证明：不碰权重也可以大幅改变行为。RAG 将外部文档动态注入上下文。核心转变：从"模型知道事实X吗？"变成了"给定事实X在上下文中，模型能用它吗？"——这就是 Norman 说的 recall→recognition。

局限：上下文窗口有限且昂贵，"lost in the middle"效应；上下文是临时的，新会话就是部分失忆。

### 第三层：能力通过基础设施（~2024-2026）
工程注意力从"我们该告诉模型什么？"转向"模型应该在一个什么样的环境中运行？"。成熟 Agent 系统越来越依赖外部记忆存储、工具注册表、协议定义、沙箱、子代理编排、压缩管线、审核循环等。

## 三个外化维度

### 1. 记忆（Memory）——外化状态
记忆系统将积累的知识（用户偏好、过往轨迹、已解决的歧义、领域事实）持久化到单个会话之外，需要时选择性检索。

架构演进：单一上下文 → 带检索存储的上下文 → 分层记忆与编排 → 自适应记忆系统

记忆作为认知工具：从 recall（回忆）变成 recognition（识别）。

### 2. 技能（Skills）——外化专业知识
技能系统将流程、最佳实践、操作指南打包成可重用的工件。Agent 不再是每次从零即兴生成行为，而是从经过验证的组件中组装行为。

三个发展阶段：原子执行原语 → 大规模原语选择 → 技能作为打包的专业知识

技能外化的五个环节：规格说明(Specification) → 发现(Discovery) → 渐进披露(Progressive Disclosure) → 执行绑定(Execution Binding) → 组合(Composition)

### 3. 协议（Protocols）——外化交互结构
协议为工具、服务和 Agent 之间定义明确的、机器可读的契约：发现、调用、委托和权限管理。

核心转变：从即兴的（ad hoc）协调变为结构化的、可治理的交换。

协议分为：Agent-工具协议（MCP/Function Calling）、Agent-Agent协议（ACP/A2P）、Agent-用户协议（AUP）等。

## Harness 工程——统一外化层

Harness 不是第四种外化，而是承载三个维度的**运行时环境**。它提供编排逻辑、约束、可观测性和反馈循环。

六个分析维度：
1. Agent 循环与控制流
2. 沙箱与执行隔离
3. 人工监督与审批关口
4. 可观测性与结构化反馈
5. 配置、权限与策略编码
6. 上下文预算管理

## 交叉分析

三个维度之间的关键耦合：
- 记忆为技能形成和协议路由提供证据
- 技能将存储的经验转化为可重用的过程，并调用协议化的行动
- 协议约束执行并将归一化结果写回记忆

参数化能力与外化能力的权衡：模型越强，对外的依赖越少；但即使最强模型（如 Claude Opus 4、GPT-5）也从结构化的外部基础设施中受益。

## 未来方向

- 自进化 Harness：系统能根据使用模式自动优化记忆结构、技能组织和协议选择
- 从私有脚手架到共享基础设施
- 如何衡量外化的有效性
- 治理与安全挑战（权限逃逸、组合风险、过度外化）

## 结语

核心洞察——Agent 进步日益取决于更好的外部认知基础设施，而不仅仅是更强的模型。这个框架解释了为什么工业界在模型上花大钱的同时，也在花更大的钱构建记忆、技能和协议系统。
