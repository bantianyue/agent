# MCP Servers - X API

Source: docs.x.com/tools/mcp

Connect AI tools to the X API using Model Context Protocol (MCP) servers for calling endpoints, searching docs, and building agent workflows.

---

## Overview: Two MCP Servers

| Server | Purpose | URL |
|--------|---------|-----|
| XMCP | Call X API endpoints (create posts, search, look up users, etc.) | `http://127.0.0.1:8000/mcp` (local) |
| Docs MCP | Search and read X API documentation | `https://docs.x.com/mcp` (hosted) |

---

## XMCP — X API Endpoints

Official MCP server that exposes X API endpoints as callable tools. Run locally and connect any MCP-compatible client (Cursor, Windsurf, custom agents).

Key Features:
- 200+ tools auto-generated from OpenAPI spec
- OAuth 1.0a authentication with browser-based consent flow
- Tool allow-listing via X_API_TOOL_ALLOWLIST
- Works with any MCP client
- Optional Grok test client using xAI API

### Quick Setup

1. Clone and install (requires Python 3.9+):
   git clone https://github.com/xdevplatform/xmcp && cd xmcp
   python -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt

2. Configure credentials:
   cp env.example .env
   Edit .env with OAuth consumer key, consumer secret, and bearer token from Developer Console (https://console.x.com). Set callback URL (e.g., http://127.0.0.1:8976/oauth/callback) and register it.

3. Start the server:
   python server.py
   Default: http://127.0.0.1:8000/mcp (configurable via env vars)

4. Connect your AI tool to http://127.0.0.1:8000/mcp

### Configuration

Add to MCP client settings:
{
  "mcpServers": {
    "xmcp": {
      "url": "http://127.0.0.1:8000/mcp"
    }
  }
}

### Tool Allow-Listing

Restrict available tools via environment variable:
X_API_TOOL_ALLOWLIST="createPosts,getUsersByUsername,searchPostsRecent,likePost"

Useful for limiting AI agent capabilities (e.g., read-only mode).

### Limitations
- No streaming or webhook endpoints — require persistent connections
- Spec fetched at startup — restart to pick up API updates
- Tokens stored in memory — not persisted across restarts

## Docs MCP — Documentation Search

Hosted at https://docs.x.com/mcp. Connect to search and read documentation without leaving your workflow.

### Available Tools

| Tool | Description |
|------|-------------|
| search_x | Search across X documentation for relevant info, code examples, API references, guides |
| get_page_x | Retrieve full content of a specific documentation page by path |

### Configuration

{
  "mcpServers": {
    "x-docs": {
      "url": "https://docs.x.com/mcp"
    }
  }
}

---

## Using Both Servers Together

Connect both simultaneously for documentation lookup AND API calls:

{
  "mcpServers": {
    "xmcp": {
      "url": "http://127.0.0.1:8000/mcp"
    },
    "x-docs": {
      "url": "https://docs.x.com/mcp"
    }
  }
}

## OpenAPI Specification

Machine-readable API spec for all X API v2 endpoints (same spec XMCP uses):

| Resource | URL |
|----------|-----|
| OpenAPI Spec (JSON) | https://api.x.com/2/openapi.json |

curl https://api.x.com/2/openapi.json -o openapi.json

Use to auto-generate API clients, import into Postman, feed into custom AI agents, or validate schemas.
