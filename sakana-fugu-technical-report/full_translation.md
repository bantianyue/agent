# Sakana Fugu 技术报告 — 全文翻译

## 摘要

前沿大语言模型（LLM）的能力持续进步，不同供应商越来越专注于不同领域。这引出一个自然而然的下一个目标：如何将多个 LLM 的个体专长组合成一个集体智能系统。为此，我们报告了 Sakana Fugu 的开发——一个编排器（orchestrator）模型家族，它利用并放大了 LLM Agent 团队的能力。Fugu 模型本身就是语言模型，经过训练以理解用户查询并动态设计 agentic scaffold 来解决这些查询。通过这些自适应 scaffold，Fugu 获得了超越任何单个 LLM Agent 的性能，在一系列具有挑战性的任务上达到了与公开可访问模型相比的 SOTA 结果，包括 SWE-Bench Pro、Terminal Bench、LiveCodeBench、GPQA-Diamond、Humanity's Last Exam 和 CharXiv Reasoning。我们发布两个变体：Fugu（在性能与延迟之间取得平衡，适合日常使用）和 Fugu-Ultra（优先在最难问题上追求答案质量）。我们描述了训练范式，包括大规模微调、进化算法和强化学习方法，以及将这些方法转化为生产系统的基础设施和核心设计原则。我们希望这份报告能鼓励对多 Agent 系统和动态的、查询自适应的 agentic scaffold 作为通向 AI 能力下一个前沿的路径进行进一步研究。

## 1 引言

前沿大语言模型现在在广泛领域达到或超越了专家级性能。然而，随着前沿能力的进步，我们也观察到越来越细粒度的模型专业化。在领域层面，过去的 GPT 系列模型通常在数学推理任务上表现 SOTA，而 Opus 系列模型在软件工程和网络安全方面有专长。在单个领域内部，差异可以更加细微：在竞赛编程中，Gemini-3.1-Pro 在直接实现已知算法方面特别有效，而 GPT 系列模型在规划和组合多个算法思路来解决最困难问题方面表现出色。这些观察表明，下一个前沿可能不是由任何单个模型实现的，而是由能够识别、组合和放大多个模型互补优势的系统来实现的。

与此同时，LLM 性能的最新提升不仅由底层模型的进步驱动，还由将 LLM 作为更大系统中一个组成部分的 agentic scaffold 驱动。这类 scaffold 用结构化提示、外部工具使用和函数调用、环境反馈以及记忆管理来增强自回归生成。这在软件工程中尤为明显，Claude Code 和其他现代 agentic scaffold 可以塑造模型的交互轨迹，提供持续的环境反馈，将原始模型能力转化为迭代 agentic 工作流。

日益多样化的语言模型和领域特定 agentic scaffold 的影响，催生了能够编排集体智能的系统的发展：动态选择涉及哪些模型、它们应如何通信、如何使用工具和与环境交互，以及何时应综合它们的输出。我们将这种模型编排视为一条新的互补扩展轴，超越了日益庞大和昂贵的语言模型。如果成功，这一方向可以使前沿能力更高效、更模块化、更广泛可及。

在这份技术报告中，我们详细介绍了 Sakana Fugu 的开发——我们的系统利用集体智能的优势来实现这一愿景。Sakana Fugu 是一个经过训练的语言模型家族，用于自适应地、动态地编排一支更强大的前沿 Agent 工作者队伍。在我们的初始发布中，我们发布两个变体。Fugu 针对速度优化，每个输入只选择一个工作者，其延迟与直接调用前沿模型相当。Fugu-Ultra 针对性能优化，为每个输入组合多个 Agent 的工作流，以额外延迟换取更高质量，面向最复杂的受益于组合多专长的任务。我们的 Fugu 模型超越了可公开访问的前沿模型，并在各基准上与 Fable 5 和 Mythos Preview 并驾齐驱，同时提供不受出口管制风险的前沿能力。

### 关于学习型编排的更广泛意义

如果能力可以通过组合现有模型来放大，而不仅仅通过训练更大模型，那么 AI 的进步就不必依赖于对最大训练运行的访问。编排提供了一条更模块化、更可及的路径，其中改进来自如何选择、协调和组合模型。这种可组合性还赋予用户对系统的有意义的控制——Agent 池可以被配置为偏好特定提供商、排除特定模型或遵守数据隐私合规约束。编排作为第一类扩展轴可能具有经济和地缘政治后果，将前沿 AI 利益更广泛分布。

## 2 相关工作

LLM 集体智能。集体智能研究相对有限的个体群体如何通过交互产生超过任何单个成员能力的行为。最近越来越多工作研究如何协调 LLM Agent 群体。Sakana Fugu 将集体呈现为单一模型接口，用户无需设计、调优多 Agent 工作流。Fugu 模型本身就是经过训练的编排器，可以为每个查询自适应地决定如何使用 Agent 池。

Agentic Scaffold。Claude Code 和 Codex 等生产系统将前沿模型包裹在丰富的交互环境中。Fugu 的方法不同：scaffold 是由训练后的编排器在推理时生成，而非应用固定交互模式。

模型合并。Sakana Fugu 是模型合并的宏观级类比。它不是在参数层面合并权重，而是在行为层面组合模型能力，将前沿模型视为黑箱 Agent，学习如何路由、协调、验证和综合它们的输出。

## 3 Sakana Fugu

Sakana Fugu 是一个学习型编排器家族，通过单一模型接口暴露多 Agent 系统。给定用户查询，Fugu 模型在一组前沿 LLM 工作者上构建 agentic scaffold，决定涉及哪些工作者、分配什么指令或角色、中间输出如何组合或验证，以及何时综合最终答案。Fugu 平衡强大性能与低延迟，适合日常交互；Fugu-Ultra 优先考虑答案质量，以额外延迟为代价使用更深编排和更大工作者池。

### 3.1 Fugu：平衡性能与延迟

Fugu 是延迟感知变体，专为响应时间与答案质量同样重要的交互式工作量设计，建立在 Trinity 基础上。

#### 参数化

Fugu 的编排器被设计为在一组前沿工作者模型上的快速决策模块。使用预训练语言模型作为骨干。给定包含 L 个工作者模型的池，在最终隐藏层后附加轻量级预测头，输出 L 个 logits 为每个模型打分。与 Trinity 不同，Fugu 始终将查询作为工作者发送给所选模型，去除角色分配将协调空间缩小到仅模型选择。

关键设计选择：Fugu 使用编排器的 logits 而非生成的文本。系统可以计算早期 token 位置的隐藏状态，应用选择头，立即分派查询——无需昂贵自回归解码。

#### 单步任务的监督微调

分两阶段训练。第一阶段：大规模 SFT。收集涵盖编码、数学、推理等的单步任务集合，每个问题可验证且有标准答案。用每个工作者模型运行 n 次，测量性能，转换为工作者上的软目标分布进行监督训练。

#### 端到端任务的进化策略

第二阶段，从真实编码助手环境收集多轮轨迹，构建端到端任务。使用 sep-CMA-ES 直接最大化编排器的期望终端奖励。这些轨迹还揭示仅凭分数看不到的 Agent 间差异——有些模型在独立基准上不突出但在交互 harness 中更鲁棒。

### 3.2 Fugu-Ultra：优先考虑性能

Fugu-Ultra 建立在 Conductor 框架之上，添加上下文适应以容纳长 horizon 函数调用和多 Agent 工作流，通过自适应 Agent 记忆。

#### 指挥一个模型管弦乐队

Conductor 利用 RL 训练语言模型对一组强大 LLM Agent 进行提示工程和协调。Conductor 输出完整 agentic 工作流作为自然语言——划分输入任务、分配任意子任务、定义目标通信策略。每个步骤指定自然语言子任务、负责的执行 Agent ID 和访问列表用于指定上下文。Fugu-Ultra 使用 GRPO 训练。

#### 函数调用 Agent 工作流

多 Agent 中任何 Agent 随时可能进行函数调用。系统必须保留哪个 Agent 发出每个调用以及其在 Conductor 工作流中的位置。工作流内 Agent 隔离防止编排崩溃——Agent 仅通过访问列表观察其他 Agent 的输出。同时允许跨工作流 Agent 间共享记忆，避免重复工具调用。

#### 训练设置

从预训练 checkpoint 扩展 Conductor 的 RL 方法。Ultra 被指示使用 Gemini-3.1-Pro、Claude-Opus-4.8 和 GPT-5.5 设计最多 5 步的 agentic 工作流。数据集为公开数据与专家设计端到端环境的混合。

## 4 能力

### 4.1 基准性能

Fugu-Ultra 在 SWE Bench Pro 达到 73.7（vs Opus 4.8 的 69.2），Terminal Bench 2.1 达到 82.1（vs GPT-5.5 的 78.2）。LiveCodeBench 上 Fugu-Ultra 93.2，LiveCodeBench Pro 90.8。GPQA Diamond 两者均达 95.5。Humanity's Last Exam 上 Fugu-Ultra 50.0。CharXiv Reasoning 86.6。

Fugu（单模型选择）也在多个基准上匹配或超越单个前沿模型——Terminal Bench 2.1 上 80.2，LiveCodeBench 92.9。

#### Agentic 编码

Fugu-Ultra 的 SWE Bench Pro 和 Terminal Bench 增益（5-6%）与整个世代提升一致。Fugu 在 Terminal Bench 中在 GPT-5.5 和 Claude-Opus-4.8 之间交替，在关键调试点调用 Opus。

#### 科学推理

两者在 GPQA-Diamond 上达 SOTA。模型识别 GPT 在数学物理的优势并针对性应用，甚至超越不公开可用的 Mythos Preview 和 Fable 5 类别——证明智能编排是无需扩展训练计算的额外性能轴。

### 4.2 领域自适应性

Fugu 模型的路由分布与 SOTA 能力先验一致。Terminal Bench 中峰值在 GPT-5.5，GPQA-Diamond 中围绕 Gemini，Humanity's Last Exam 中三个 Agent 高度平衡分布。

### 4.3 超越基准的性能

AutoResearch 训练优化：Fugu-Ultra 123 次实验后平均验证 BPB 0.9774，领先所有单 Agent 基线。

古典日语阅读顺序：Fugu-Ultra 平均 NED 0.776，领先所有前沿模型（最佳基线 0.642）。

CAD 生成：Fugu-Ultra 生成结构完整的光圈机构，其他模型不够完整。

### 4.4 最优策略和拓扑

辩论与聚合：Fugu-Ultra 按问题动态选择聚合器——知识密集型任务用 Gemini 聚合，数学密集任务用 GPT 聚合——正是现有多 Agent 系统缺少的自适应能力。

构建与调试：GPT 作为构建者，Opus 在关键时刻做质量验证。在 PyPI 服务器构建中 Opus 发现了 GPT 实现中的三个缺失问题。

引入专家：在密码分析任务中 Opus 做第一版攻击，GPT 从基本原理重新推导整攻击。

## 5 结论

Sakana Fugu 证明了模型编排可以作为前沿 AI 能力的实用扩展轴。因为它在行为层面而非参数层面组合模型，可以纳入新工作者模型、适应不同用户约束、支持更广泛的部署和隐私要求。Fugu 指向一种新的 AI 进步观——能力不仅来自扩展单个模型，也来自学习专门模型如何协作。
