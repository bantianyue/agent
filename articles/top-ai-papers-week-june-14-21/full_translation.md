# 本周 Top AI 论文（6月14日 - 6月21日）

## 1. SpatialClaw

3D 和 4D 场景的空间推理仍然是通用视觉语言模型的短板，因为它们直接输出文本答案而非实际测量。来自 NVIDIA 的 SpatialClaw 是一个无需训练的框架，重新思考了动作接口，让 VLM 驱动的 Agent 通过代码进行推理。Agent 每步向一个持久的 Jupyter 内核写入一个 Python cell，该内核预加载了感知基元和科学库，然后检查中间结果并在各步之间修正策略。

- 代码作为动作接口：SAM3 分割、Depth-Anything-3 重建和几何工具等感知工具作为普通 Python 可调用函数暴露，Agent 以编程方式组合它们，而非从像素猜测空间关系。
- 持久有状态内核：掩码、深度图、相机几何和轨迹是内核在各轮之间保留的普通 Python 变量，因此某一步生成的任何对象在后续步骤中仍可用于组合、检查和修正。
- 无需适配的强劲结果：在涵盖静态和动态任务的 20 个空间推理基准上，SpatialClaw 达到 59.9% 的平均准确率，超过此前最好的空间 Agent 11.2 个百分点，在两个模型家族的六个 VLM 主干上表现一致。
- 为什么重要：由于无需训练且与模型无关，SpatialClaw 将代码执行变成了空间推理的通用基底，任何有能力的 VLM 都可以接入，而不需要专门的空间微调。

## 2. Compositional Skill Routing（组合式技能路由）

真实任务很少映射到单个技能。它们通常需要多个技能组合在一起，但大多数技能路由仍然将问题视为从库中挑选一个工具。这项工作形式化了组合式技能路由（Compositional Skill Routing），即 Agent 必须从大型库中选择并排序多个可复用技能以满足复杂查询，并引入了 SkillWeaver——一个围绕它构建的分解-检索-组合流水线。

- 三阶段流水线：SkillWeaver 用 LLM 将查询分解为子任务，用双编码器加 FAISS 索引将每个子任务匹配到一个技能，然后执行依赖感知规划以组装可执行的计划。
- 真实基准：作者发布了 CompSkillBench，一个包含 300 个组合查询的基准，覆盖 2,209 个真实 MCP 服务器技能，横跨 24 个功能类别，因此路由测试是在实际工具生态上进行的，而非玩具库。
- 分解是瓶颈：任务分解质量成为主要限制因素，迭代式技能感知分解（将检索信息反馈回分解步骤）将准确率从 51.0% 提升到 67.7%。
- 为什么重要：随着 Agent 技能库扩展到数千个条目，单工具路由不再够用，将路由视为组合规划问题才能让 Agent 处理真正的多步请求。

## 3. PreAct

计算机使用 Agent 通过屏幕驱动真实软件，但它们每个任务都从头解决。让它们重复一个任务，它会重新读取屏幕并重新推理每一次点击，再次付出全部成本。PreAct 通过将首次成功运行编译成一个小的状态机程序来解决这个问题——状态检查屏幕，转换执行动作——然后在后续运行中重放该程序，而不是调用 Agent。

- 将运行编译为状态机：完成的任务被捕获为显式程序而非自由形式的轨迹，将一次性解决方案变成可确定性执行的可复用制品。
- 重放无需每步模型调用：重放编译后的程序比 Agent 快 8.5 到 13 倍，因为在重复任务上不需要每步的语言模型调用。
- 构造安全：每一步 PreAct 都会检查屏幕是否与程序预期匹配，然后才执行动作，一旦出现偏差就将控制权交还给 Agent，并且只存储独立评估者确认能从干净状态解决问题的程序。
- 为什么重要：这将计算机使用 Agent 从重新推理一切的交互式工具变成了可重复的操作系统，这正是将它们部署到重复性实际工作中所需要的。

## 4. Can LLM Agents Infer World Models?（LLM Agent 能推断世界模型吗？）

LLM Agent 真的能构建一个它看不到的环境模型吗？这项工作通过 Agent 式自动机学习使这个问题变得可评分。Agent 必须通过两个接口与预言机交互来发现隐藏的确定性有限自动机：成员查询（询问某个字符串是否属于目标语言）和等价查询（询问提议的自动机是否正确），这为交互式发现提供了一个干净、可扩展的测试平台。

- 可评分的世界模型测试：将世界模型推断视为 DFA 学习，给出了客观的成功标准和可衡量的交互效率，以经典的自动机学习算法作为强基线。
- 可控、可扩展的难度：隐藏自动机的大小充当难度旋钮，因此基准可以平滑地扩展任务复杂度，而不是依赖一组固定的谜题。
- Agent 落后于经典算法：当前 Agent 有时能执行非平凡的交互式发现，但性能随着 DFA 规模增长急剧下降，轨迹分析揭示了查询规划、证据整合和假设构建中的反复失败。
- 为什么重要：推理模型在这里明显优于非推理模型，但与经典算法之间的巨大差距表明，系统性的交互式世界模型构建仍然是一个未解决的能力，而非规模化的副产品。

## 5. From Trainee to Trainer（从受训者到训练者）

谁应该为 RL Agent 设计训练环境——实践者还是策略本身？LLM 的 RL 流水线通常依赖各阶段之间手动重新设计的环境，实践者猜测哪种配置最能改进当前策略。这篇论文将这项工作交给了模型，提出了 LLM-as-Environment-Engineer 框架，其中策略诊断自己的弱点并提出下一个要训练的环境。

- 策略设计自己的课程：不是在阶段之间由人类重塑环境，当前策略分析失败轨迹并结合上下文信息，提出对下一阶段训练环境配置的修改。
- 失败驱动的环境编辑：由于提议基于策略的实际失败模式，课程针对的是阻碍模型前进的具体缺口，而非通用的难度提升。
- 受训者成为训练者：一个关键发现是，当前的 RL checkpoint 比原始基础模型更适合作为环境工程师，这表明学会行动也提升了模型诊断自己还不会什么的能力。
- 为什么重要：手动阶段间环境设计是 LLM 的 RL 中最不可扩展的环节之一，让策略引导自己的课程关闭了一个拖累 Agent RL 的慢速人工循环。

## 6. OpenClaw-Skill

为 LLM Agent 配备有效技能是实际系统中的大部分工作，但大多数技能归纳工作一次只蒸馏一条轨迹，这产生了狭窄、脆弱的技能。OpenClaw-Skill 引入了集体技能树搜索（Collective Skill Tree Search, CSTS），一个基于树搜索的技能构建框架，构建结构化的、多样化的、可泛化的技能树，然后训练 Agent 实际使用它构建的内容。

- 集体技能树搜索：CSTS 不是将单条轨迹蒸馏成单个技能，而是在候选技能树上进行搜索，使用多个模型生成和评估它们，使库捕获多样化的策略。
- 结构化的可复用技能树：分层组织技能产生跨工具使用、多步推理和环境交互的泛化能力，而非过拟合到单个任务。
- 训练 Agent 利用技能：构建树只是工作的一半，因此该框架将构建与一个学习步骤配对，教导 Agent 有效检索和应用构建的技能层级。
- 为什么重要：可复用的技能库正在成为有能力的 Agent 的骨干，从逐轨迹蒸馏转向集体树搜索是一个具体的配方，用于构建在任务增长时仍然有用的库。

## 7. Back on Track（PAPO）

扩散大语言模型以不适合为自回归模型构建的强化学习配方的方式生成文本，训练它们推理暴露了两个具体问题。奖励是稀疏的，单一终端奖励无法指导中间生成步骤；策略更新有时会漂移到不自然的轨迹而非真实的生成路径。这篇论文提出了过程对齐策略优化（Process Aligned Policy Optimization, PAPO）来修复这两个问题。

- 命名了两个失败模式：这项工作将稀疏奖励和轨迹漂移确定为扩散 LLM 推理中稳定 RL 训练的核心障碍，而非将训练不稳定性视为黑箱。
- 步感知过程奖励：PAPO 将终端奖励转化为细粒度的步级指导，使中间去噪步骤收到学习信号，而不是等待单一的序列结束分数。
- 熵引导的重演：在关键的高不确定性时刻，该方法重放真实的生成路径，使更新与模型实际产生文本的方式保持一致，而非追逐人工轨迹。
- 为什么重要：扩散 LLM 是自回归模型的一个严肃替代方案，为它们提供稳定的推理 RL 配方——在 GSM8K 和 MATH500 等基准上报告了 4.5% 到 42.2% 的提升——有助于缩小两种范式之间的推理差距。

## 8. AtomMem

LLM Agent 的长期记忆通常在两个方面失败：粗粒度的摘要随时间漂移，不受约束的更新会破坏已存储的内容。AtomMem 将记忆单元保持得很小，使用一个事实执行器（Fact Executor）从长交互中选择性地提取高价值原子事实，并将它们组织成层级事件结构和时间用户画像，带有一个在检索时重新连接碎片化记忆的关联记忆图。该方法在 LoCoMo 长期记忆基准上报告了最先进的结果。

## 9. Beyond Domains（SkillMigrator）

LLM 网页 Agent 通常作为工具调用者运行，每轮读取一个新页面并发出一个低级动作，因此任务视野和 LLM 补全次数都会膨胀。这项工作使网页技能跨站点可复用，通过 SkillMigrator 将归纳的技能存储为以页面布局结构而非指令相似性或站点元数据为键的可迁移交互模式，因此在一个站点上学习的技能会在具有相同交互形状的新站点上触发。它在 WebArena 和 Mind2Web 上将平均 LLM 动作次数减少了 8% 到 10%，成功率相当。

## 10. The Stanford EDGAR Filings Dataset

干净的长上下文文档在预训练中仍然稀缺，尤其是在金融领域。这个发布将美国 SEC 公司财务和披露文件重建为布局忠实、token 高效的多标记 Markdown（MultiMarkdown），在 SEFD-v1 中发布了 152B tokens，来自一个估计 550B token 的档案库，涵盖 1,850 万份文件，与 Common Crawl 语料库的重叠率低于 0.1%。它还附带两个衍生基准：EDGAR-Forecast（用于数值预测）和 EDGAR-OCR（用于金融表格转录），以支持金融推理、预测和文档理解。
