现在每天早上，是我的 Agent 而非我在跑我全部五家生意，而且我不打算回去了。

我同时经营五件事。一家 MVP 代工机构 @ignytlabs（月经常性收入 $20K），通过 @aimvpbuilders 教了 3500 多人用 AI 做软件，一个移动端应用工作室（已有几个上线的应用），面向创始人的一对一咨询，还有一个 5 万粉丝的 Instagram（跑赞助内容）。

几周前我写过一篇 Hermes Agent 的完整分析——为什么它是我第一个真正信任、能在后台运行工作而不仅仅是回答问题 AI Agent。如果你还没读过那篇，先从那篇开始看完整配置。链接在这里。

这篇文章要讲的是上周发生了什么事。Nous Research 发布了一个大更新，这是 Hermes 从我边测试的工具变成真正跑我每天大量工作的原因。

AlexFinn 在推文里拆解了这个更新的所有内容。我把他的清单拿过来，在我所有生意上全设了一遍。下面讲的是实际发生了什么变化，以及每个功能我是怎么用的。

---

如果你是第一次接触 Hermes，快速回顾一下。绝大多数你用的 Agent 是基于 session 的。你打开 Claude Code 或 Cursor，交给它一个任务，关掉标签页，第二天早上它就把你和你的工作全忘了。Hermes 是反过来的。它跑在服务器上，全天都在线，记住每一段对话，并在运行过程中自己写技能文件。你用的时间越长，需要解释的东西就越少。

就凭这一点，在我写那篇文章时它就已经是另一个类别的工具了。上周的更新让它的整体体验变得足够好，以至于我真的把运营任务搬了上去。

---

你现在可以把 Hermes 连接到原生 iMessage。设置免费，这意味着这个 Agent 出现在我和团队、家人聊天的同一个应用里。不需要打开单独的 App，不需要登录仪表板。我就像跟人发短信一样跟它说话。

我的用法很简单。不在工位时用 iMessage 处理快速事务。在电脑前做专注工作时用桌面 App。无论是哪种方式，都是同一个 Agent、同一套记忆。

连接只花了几分钟。最简便的方式是 Photon，它会给你一个托管的 iMessage 线路，不需要自己维护一台 Mac。

→ 在终端跑 `hermes photon setup --phone +1...`
→ 在浏览器中确认设备登录
→ 它分配一个号码，告诉你用哪个号码发短信就能联系到你的 Agent

整个设置就这些。如果你已经有 Mac 常开，也可以用 BlueBubbles 方案，但 Photon 是不需要硬件的入门方式。

想想这对你与工具的关系意味着什么。以前，我的 AI 藏在一个需要我主动去找的登录页后面。现在它就在我口袋里，和我其他所有人的消息在同一个收件箱里。

听起来很小，但真的开始这么做之后就完全不一样了。你会在想法冒出来的瞬间就把工作交给它——比如在两个咨询之间想起一个跟进方案——而不是等到回到笔记本前再处理。

---

后台 Agent 现在默认开启。当你交给 Hermes 一个复杂任务时，它会安静地启动子 Agent 在后台运行。你不需要坐在那里看着转圈。你发出任务后可以继续跟它聊别的，重活在后台完成。

以前用任何 Agent 的方式都是：给任务、等、跟进、再等。新的默认行为反过来。你一边跟 Agent 聊天，它一边已经在跑你之前交给它的四件事。

对代工机构来说，这是助理和正式员工之间的区别。一个团队里的初级成员不会每次你交给任务就把整个对话卡住。他们会去做然后回来汇报。Hermes 现在也是这样工作的。这一个默认行为的改变，让我放心地把客户调研和初稿工作交给了它。瓶颈不再是「我要等多久」，而是「我能在把账算丢之前同时跑多少个」。

---

桌面 App 拿到了一批更新，加起来变化不小。

→ 可以把聊天窗口弹出来变成独立窗口，这样我可以一边看一个 Agent 工作一边在另一个窗口干活
→ 模型选择器移到了底部——你真正会去够的位置
→ 有了实时的子 Agent 面板，可以实时看到后台 Agent 在做什么
→ 内置了终端，不需要在 Hermes 和独立终端窗口之间来回切换

这些功能单独看都不是头条更新。但它们合在一起，把桌面 App 从一个聊天框变成了更像控制室的东西。那个实时的子 Agent 面板是我整天都开着的。当后台 Agent 在跑客户工作时，能看到实际的工作进行而不是靠猜，这才让我放心离开去接咨询电话。

---

Profile 是大多数人都跳过的功能，这次更新让它们变得不能忽视。

Hermes 里的 Profile 本质上是一个与其他 Profile 并行的独立 Agent，有自己的记忆和技能。你现在可以直接从浏览器创建。在终端打 `hermes dashboard`，进入 Profiles，几分钟就能设好。

这就是我实际用一个工具管理五家生意的方式。我设置了四个 Profile，对应我想克隆自己时需要雇的团队：

→ 一个 Chief of Staff Profile，了解我在五个生意中的优先事项，跑早上简报
→ 一个 Agency Profile，处理客户调研和 MVP 项目的初稿需求文档
→ 一个 Content Profile，撰写 X 文章草稿、推文和赞助内容初稿
→ 一个 Community Profile，持续感知社区的提问并提炼主题

还有一件值得偷师的事：即使你只有一个需求，也至少跑两个 Profile 以保证可靠性。一个 Profile 是单点故障。两个就是互相照应的团队。

---

这一项悄悄地修复了第一个版本中最大的摩擦点。

Hermes 在运行过程中自己写技能，这很好，但之前没有简便的方式获取别人已经建好的技能。新的 Skills Hub 就在仪表板里，让你可以浏览并一键安装热门技能。

以前，每个新 Profile 从零开始，需要几周才能建立起有用的技能库。现在你可以在第一天就用经过验证的技能来播种一个新 Profile，然后让它从那里开始专业化。我创建我的 Content Profile 时，这就让我在第一天就植入经过验证的技能，而不是从零开始建库。

---

自改进循环在这次更新中变得更聪明了。

你的 Agent 现在会在你工作过程中更积极地编写和更新自己的技能，而且对自己的记忆所做的编辑质量更高。Hermes 最初的定位就是「用得越久越好」。这次更新加速了这个曲线。我的 Agency Profile 在每次客户构建后已经会自动调整需求文档的写法。整个思路就是这种边际效应在运行中不断累积。

这是其他工具真正难以复制的部分，也是我一直在跟社区说 Hermes 是另一个类别的原因。大多数 AI 工具在第 90 天和第一天一样好。而 Hermes 应该是可测量地更好——因为它一直在自己写自己的操作手册。

---

这是最炫酷的更新，也是大多数人不会碰的部分，所以我直说了。

Hermes 现在有了 Unreal Engine 5.8 MCP。Unreal 是地球上最强大的游戏引擎，大量商业游戏都基于它。装上 MCP 后，你可以让 Agent 去建复杂的 3D 游戏。

我不打算假装下周就开一家游戏工作室。对大多数开发者来说这只是炫技，不是日常工具。但它作为一个信号很重要。Hermes 周围的 MCP 生态正在扩展到严肃的专业软件，不只是小网页工具。当一个 Agent 能驱动 Unreal 时，问题不再是「它能建什么」，而是「它连不了什么」。对我的应用工作室来说，这是我正在密切关注的方向。

---

最后一个，快速的。

Hermes 现在通过 Bot API 10.1 在 Telegram 中发送正确的富消息，有干净的排版和标记，而不是一堵纯文本墙。如果你在移动中做更深入的工作，这是真正的升级。

我现在的分工是：行动中用 iMessage 处理快速提示，离开办公桌做专注工作但希望 Agent 的输出真正可读时用 Telegram。

---

把所有这些放在一起，画面就很清楚了。

我写第一篇分析时，Hermes 是一个强大的 Agent，我在机器上驱动它。这次更新之后，它是一个我随时随地都能接触到的 Agent，默认在后台运行工作，我可以像一个小团队一样用多 Profile 而非单一个体来运作，而且在我睡觉时还会自我提升。

这不是一个新工具。它是同一个工具长大了。而且整个设置仍然只需要大约 30 分钟。

---

在你全情投入之前，先说几个实话的警示。

后台 Agent 默认开启很好，直到你忘了它们消耗 Token。你启动的每个子 Agent 都是实打实的费用。第一个星期盯一下用量，这样在账单吓到你之前你就知道自己的使用模式了。

至少跑两个 Profile，但不要第一天就建十个。每个 Profile 需要真正的工作。空的 Profile 只会弄乱仪表板和自己的工作流程。

Browser 和 Computer Use 出于安全考虑仍然默认关闭，你应该保持这个状态，直到某个 Profile 真的需要它们。不要把你栈里的每个 Agent 都交到你登录的任何东西的关键权限。

Unreal MCP 很令人印象深刻，但还在早期。在你看到它在真实场景中撑住之前，把它当作实验，而不是生产流水线。

---

基于 session 的 Agent 和持久化 Agent 之间的差距变得更大了。

大多数开发者仍然活在一个「打开标签页→做任务→关标签页」的世界里。这对专注编码没问题。但栈的另一半——全天运行、记住一切、给你发短信回来、自我改进的系统——几乎没有人在用。

对我来说，这属于 AI 员工这个类别。代工机构跑着 $20K 月经常性收入，其中很大一部分工作——构建、研究、初稿、社区脉搏、赞助内容初稿——现在都正在迁移到上面说的那些 Profile 上。这次更新后的 Hermes，是我用过的最接近于一组「我不在时也醒着」的员工。

今年把这些搭建起来的开发者，明年会看起来不公平地领先。他们只是每天扔掉了向一个每天忘记他们的工具重新解释上下文的税。

更新已经上线。设置需要 30 分钟。复利是永久的。

我们还太早了。
