要点速览 - 半自回归架构：将并行骨干网络与轻量级顺序头结合，保持并行草稿速度的同时缓解后缀衰减，接受长度相比 Eagle3 提升 26.7-30.9%，相比 DFlash 提升 16.3-18.4% - 置信度调度验证：通过时序温度缩放校准后，硬件感知调度器动态修剪低置信度后缀 token：对话任务接受率从 45.7% 升到 95.7%，数学从 76.9% 升到 92.5% - 生产结果：在 DeepSeek-V4 真实流量中每用户生成速度加速 60-85%（Flash）和 57-78%（Pro），严格 SLA 下避免吞吐悬崖式下跌 - 开源：DSpark 检查点与 DeepSpec 训练仓库（含 Eagle3、DFlash、DSpark）全部开源 推测解码通过将草稿生成与目标验证分离来加速大语言模型推理。一个轻量级草案模型提出候选 token 块，完整的语言模型通过拒绝采样在单次前向传播中验证整个块，接受与目标分布一致的最长前缀并附加一个奖励 token。因为验证是并行的且接受规则精确保留了目标分布，推测解码在不损失质量的前提下加速了生成。然而，草稿模型的设计始终在生成速度和质量之间做权衡。自回归草案模型每个 token 以前面采样的 token 为条件，质量高但生成速度随块大小线性增长，只能用浅层和小块。并行草案模型所有 token 一次前向生成，速度几乎不随块大小变化，但每个位置独立预测，到后半段会出现多模态碰撞，接受率急剧下降。DSpark 由北京大学和 DeepSeek-AI 联合提出，其思路是：同时解决后缀衰减和验证浪费两个瓶颈。草稿模型架构的权衡并行草案模型在单次前向传播中产生所有 γ 个草稿 token，使 T_draft 几乎与块大小无关。其中 DFlash 是当前 SOTA 方案，其草案模型以目标模型提取的丰富上下文特征（KV 注入）为条件。预填充期间，来自一组目标层的隐藏状态被拼接并投影到草案隐藏空间，然后在每层中沿序列维度与草案块表示拼接。所有块内位置彼此双向关注并关注注入的目标上下文。因为草案仅需单次前向，DFlash 在相同延迟预算下可以使用比自回归草案模型更深层的架构和更大的块。但大并行草稿块有两个问题要解决。第一，并行草案模型无法建模块内 token 间依赖，导致衰减。第二，不加区分地验证所有提议 token 在高并发下会降低系统吞吐：验证额外 token 在轻负载下免费，在重负载下却占用关键批容量。半自回归架构：并行速写×顺序纠偏图 1 | DSpark 架构。并行骨干网络生成草稿 token，顺序头注入依赖关系，置信度头预测存活概率，硬件感知调度器决定哪些 token 被验证。DSpark 将草稿生成分为两个阶段。并行阶段：基于 DFlash 的并行骨干网络在整个块上运行单次前向传播，产生隐藏状态和基础 logits。改动在于：不再使用锚点 token 加 γ 个 mask token 并只预测 mask 位置，而是将锚点本身作为第一个预测位置，所以 γ 个输入 token（锚点 + γ-1 个 mask）就产生 γ 个草稿 logits。同等质量下计算更少。顺序阶段：为基础 logits 补充前缀相关的转移偏置 B_k(x0, x<k, xk)，允许每个草稿位置以块内先前采样的 token 为条件。这通过自回归分解诱导出一个因果块分布：p_k(v | x0, x<k) = exp(U_k(v) + B_k(x0, x<k, v)) / Σ exp(U_k(u) + B_k(x0, x<k, u))其中 x0 是上一轮验证周期产生的锚点 token，U_k 是并行骨干网络在位置 k 产生的基础 logit 向量。推理时顺序块从左到右按 p_k 采样。因为采样过程本质上是顺序的，顺序块必须计算轻量（T_sequential ≪ T_parallel），使整体草稿延迟仍由并行阶段主导。两个实现，越简单越好。马尔可夫头：B_k 只依赖紧邻前一个 token，简化为低秩矩阵 B = W1W2（r=256）。W1 是嵌入查找表，W2 是 logit 投影。一旦位置 1 采样到 "of"，马尔可夫头就在位置 2 上提升 "course" 并抑制 "problem"。参数极省、计算极快。RNN 头：通过循环状态 s_k 累积块内的完整前缀历史。每一步将当前状态 s_{k-1}、前一 token 嵌入和骨干隐藏 h_k 拼接为输入向量 z_k，然后应用门控更新：s_k = σ(W_g z_k) ⊙ s_{k-1} + (1 - σ(W_g z_k)) ⊙ tanh(W_c z_k)。比马尔可夫头更强，但实践中的增量收益有限。默认用马尔可夫头。置信度调度验证产生更多草稿不等于更高加速。在低并发下验证拒绝的 token 几乎免费，但在高并发下每个不必要的验证都在消耗关键批容量。DSpark 的解决方案是置信度调度验证，两个组件。置信度头：为每个草稿位置 k 输出 c_k ∈ (0, 1)，建模在该块中所有前面 token 被接受的条件下该 token 能通过目标验证的条件概率。架构是轻量级线性投影 + sigmoid：c_k = σ(w^T [h_k; W1[x_{k-1}]])。训练用分析接受率 c_k 监督，后者由草案分布与目标分布的总变分距离决定：c_k = 1 - ½‖p^d_k - p^t_k‖₁。但神经网络置信度估计过度自信。直接用原始分数会扭曲吞吐估计。论文提出时序温度缩放：由于每个 c_i 是条件概率，前缀存活概率 = Π c_i。在留出验证集上从左到右校准这个累积乘积，每位置执行一维网格搜索使期望校准误差最小。这是顺序保持变换，不改变相对排序，只是把估计校准到真实接受率上。校准将平均 ECE 从 3-8% 降到了约 1%。硬件感知前缀调度器将验证长度选择制定为全局吞吐最大化问题。对 R 个请求的批次，存活概率 a_{r,j} = Π c_{r,i}。设验证步中总批大小 B = Σ (1 + ℓ_r)，期望接受 token 数 τ = Σ (1 + Σ a_{r,j})。SPS(B) 是引擎吞吐（每秒步数），在引擎初始化时预配置。调度器最大化 Θ = τ · SPS(B)。算法全局排序所有有效前缀扩展，增量式从排序池中接纳 token，通过成本表的 O(1) 查找更新吞吐。为严格执行因果性，调度器采用早停：吞吐下降就停，裁剪决策只依赖于已处理的前缀。在真实部署中调整为异步调度，使用两步前的置信度头输出近似即将到来的验证容量，形成因果屏障。训练目标训练期间目标模型完全冻结，草案模型共享其嵌入层和语言模型头（也是冻结的），只更新骨干草案器、顺序块和置信度头。训练损失三项：• 交叉熵 L_ce：训练草案器预测正确下一个 token• 总变分距离 L_tv：惩罚草案与目标分布偏差，直接最大化期望接受率• 置信度 BCE L_conf：训练置信度头预测软接受标签全部按位置加权 w_k = exp(-(k-1)/γ)，更看重靠前的块位置。整体 L = 0.1L_ce + 0.9L_tv + 1.0L_conf。分布匹配损失占训练主导，说明训练对目标模型输出分布准确性很敏感。离线实验评估用四种目标模型：Qwen3-{4B, 8B, 14B} 和 Gemma4-12B。草案比较对象：Eagle3（自回归，TTT，1 层）和 DFlash（并行，5 层）。训练数据 Open-PerfectBlend 2（130 万样本）。评估覆盖数学推理（GSM8K、MATH500、AIME25）、代码生成（MBPP、HumanEval、Live-CodeBench）、日常对话（MT-Bench、Alpaca、Arena-Hard）。表 1 | DSpark
