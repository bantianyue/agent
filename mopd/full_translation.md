# MOPD：面向 LLM 后训练的多教师同策略蒸馏能力集成

作者：Wenhan Ma（北京大学/小米实习）, Jianyu Wei, Liang Zhao, Hailin Zhang, Bangjun Xiao, Lei Li, Qibin Yang, Bofei Gao, Yudong Wang, Rang Li, Jinhao Dong, Zhifang Sui, Fuli Luo（北大/LLM Core）

## 摘要

现代大语言模型（LLM）在后训练阶段依赖强化学习来提升特定能力，但将多种能力集成到一个模型中仍然很困难。现有方法如 Off-Policy Finetune 和 Mix-RL 要么效率低下，要么损失性能。本文提出 MOPD（Multi-Teacher On-Policy Distillation，多教师同策略蒸馏），一种新的后训练范式：先对每个领域进行专门的 RL 训练获得领域教师，然后在学生自己的 rollout 上将这些教师蒸馏到学生模型中。这消除了暴露偏差（exposure bias）并提供了密集的优化信号。在 Qwen3-30B-A3B 上，MOPD 优于 Mix-RL、Cascade RL、Off-Policy Finetune 和 Param-Merge 基线，几乎继承了每位教师的全部能力。MOPD 还实现了领域教师的并行独立开发，消除了多领域后训练中典型的跨领域耦合。MOPD 已部署在 MiMo-V2-Flash（一个工业级前沿模型）的后训练中，展示了其在超大模型能力集成中的实用价值。

## 1 引言

强化学习已成为提升 LLM 能力的重要方法。不同的任务领域依赖不同的 RL 流水线：数学推理用可验证答案的 RL，软件工程用可执行沙箱中的 Agent 风格 RL，指令遵循和创意写作用基于评分标准（rubric）的 RL，搜索 Agent 用网络环境 RL。每条流水线都可靠地提升了模型在目标领域的能力。但我们最终需要的是一个在所有领域都表现良好的单一模型。构建这样的模型仍然是现代 LLM 后训练中的一个开放问题。

**Table 1：能力集成范式在三个实际维度上的对比。MOPD 是唯一同时实现密集优化、同策略训练和可并行流水线的方法。**

| 方法 | 密集优化 | 同策略 | 可并行 |
|------|---------|--------|-------|
| Param-Merge | ✗ | — | ✓ |
| Off-Policy Finetune | ✓ | ✗ | ✓ |
| Mix-RL | ✗ | ✓ | ✗ |
| Cascade RL | ✗ | ✓ | ✗ |
| MOPD (ours) | ✓ | ✓ | ✓ |

现有方法分为四类：
1. **Mix-RL**：将各领域提示混合后联合 RL，但跨领域训练信号相互干扰，产生跷跷板效应（seesaw effect）。
2. **Cascade RL**：顺序训练各领域，早期领域能力可能被覆盖，长时间运行放大稳定性风险。
3. **Off-Policy Finetune**：训练各领域 RL 教师后在学生在教师 rollout 上做 SFT，暴露偏差导致性能下降。
4. **Param-Merge**：在权重空间平均或组合任务向量，融合后的模型通常不稳定。

MOPD 直接在策略空间中做多教师能力集成，而非权重空间或静态数据集空间。先对每个领域独立做 RL 获得教师，再将教师同策略蒸馏到学生中。三方面结构优势：密集训练信号、消除暴露偏差、策略空间路由而非权重融合。

在 Qwen3-30B-A3B 上 MOPD 领先最佳基线 5.5 个归一化得分点（0.937 vs 0.882），并在工业级模型 MiMo-V2-Flash 上验证了实际效果。

## 2 相关工作

### 2.1 LLM 的强化学习

RLVR（带可验证奖励的 RL）中，PPO 和 GRPO 风格的结果奖励优化已成为标准实践。RL 流水线涵盖数学推理、软件工程、指令遵循和搜索 Agent。每条流水线产生一个领域专家，将它们整合为单一模型自然引出了多领域/多任务问题。

### 2.2 LLM 的蒸馏

经典蒸馏在教师完成文本的固定语料上最小化前向 KL，是异策略（off-policy）的。同策略蒸馏在学生自己的 rollout 上用教师作为评分信号训练，如 MiniLLM（反向 KL）和 Teacher-as-reward 方法。MOPD 将"学生采样、教师评分"模板扩展到多个按提示路由的领域教师。

### 2.3 模型合并

模型合并在不额外训练的情况下在权重空间融合多个 checkpoint。Model Soups 平均独立微调模型的权重，Task Arithmetic 通过加减任务向量组合能力。后续工作用 TIES-Merging、DARE、AdaMerging 缓解参数冲突。

## 3 方法

### 3.1 流水线

MOPD 将后训练组织为三个阶段：

- **Stage 1: General SFT**。在覆盖所有基础能力的广泛语料上微调基础模型，得到 SFT checkpoint。这是 Stage 2 领域 RL 的共享初始化，也是 Stage 3 MOPD 学生的初始化。

- **Stage 2: Domain-specialised RL Training**。对每个领域 d，从 Stage-1 checkpoint 开始训练领域专家。每个专家用最适合其领域的 RL 方案（数学用可验证答案的 RL，软件工程用可执行沙箱 Agent RL）。各领域训练完全并行。

- **Stage 3: MOPD**。学生从 Stage-1 SFT checkpoint 初始化，Stage-2 领域专家冻结形成教师组。在多领域数据集上训练，每个优化步骤：采样 prompt 批次 → 学生生成轨迹并记录每 token 概率 → 按领域分发给对应教师获得每 token 概率 → 最小化学生与教师间的每 token 反向 KL。

MOPD 的结构优势：
1. **无暴露偏差**：学生在自己的 rollout 上训练
2. **密集的每 token 监督**：教师在每个 token 提供概率分布
3. **策略空间中的稳定集成**：按提示路由而非权重融合
4. **模块化并行教师**：各教师完全并行，独立调参
5. **同源教师稳定性**：教师和学生共享相近的策略分布

### 3.2 算法

学生在 Stage 3 通过每 token 反向 KL 优化，目标函数为：
L_revKL = E[(1/|y|) Σ_t Σ_v π_θ(v) log(π_θ(v)/π_φ_d(v))]

#### 3.2.1 Policy-Gradient 实现

遵循 MiniLLM 将蒸馏视为 RL 过程。梯度中教师-学生的 log-difference 充当每 token advantage，用双侧裁剪保持稳定。

#### 3.2.2 Top-k 蒸馏实现

在教师 top-k token 上蒸馏，利用更多教师分布信息。加入修正项确保 top-k 截断不影响最优解。同时减少基础设施负担（教师不需要传输完整词汇分布）。

### 3.3 基础设施

Stage 3 的核心额外操作是教师预填充（teacher prefill）。将每个领域教师部署为独立预填充服务，学生 rollout 完成时异步发送请求。教师预填充与下一批采样重叠，几乎不增加墙钟时间。

## 4 实验

在 Qwen3-30B-A3B 上比较 MOPD 与四种能力集成范式，并在 MiMo-V2-Flash（309B 参数）上验证。

### 4.1 实验设置

基础模型：Qwen3-30B-A3B-Base，SFT 后作为共同起点。三个领域：数学（AIME25/AIME26）、指令遵循（IFBench/IFEval）、软件工程（SWE-bench Verified）。

五种基线：Mix-RL、Cascade RL、Off-Policy Finetune、Param-Merge（线性平均/Task Arithmetic）、MOPD。

使用归一化得分消除不同领域绝对空间差异，0=Stage-1 SFT 学生，1=领域特化教师。

### 4.2 主要结果

**Table 2: Qwen3-30B-A3B 多领域能力集成结果**

| 方法 | AIME25 | AIME26 | IFBench | IFEval | SWE-bench V. | 归一化得分 |
|------|--------|--------|---------|--------|-------------|-----------|
| Student (SFT) | 45.42 | 54.48 | 42.69 | 84.17 | 35.80 | 0.000 |
| RL Teacher | 54.79 | 63.65 | 78.40 | 95.50 | 51.20 | 1.000 |
| Mix-RL | 52.71 | 63.75 | 75.00 | 94.58 | 48.80 | 0.882 |
| Cascade RL | 48.54 | 61.88 | 77.11 | 95.80 | 47.80 | 0.775 |
| Off-Policy Finetune | 51.56 | 63.44 | 80.95 | 93.35 | 45.80 | 0.824 |
| Param-Merge (Avg) | 47.81 | 59.58 | 53.74 | 88.79 | 39.60 | 0.328 |
| Param-Merge (Task Arith) | 49.38 | 63.96 | 78.23 | 95.81 | 48.80 | 0.857 |
| **MOPD (ours)** | **51.46** | **65.31** | **77.89** | **93.84** | **50.40** | **0.937** |

四个关键观察：
1. Mix-RL、Cascade RL、Off-Policy Finetune 各有领域偏差，归一化得分 0.775-0.882 相近但有不同弱点
2. MOPD 达到最高归一化得分 0.937，且三个领域归一化范围仅 0.044，是所有方法中最均匀的
3. Param-Merge 高度依赖合并方法，线性平均仅 0.328
4. MOPD 样本效率更高，在 IF 上约 25K 样本达到教师级，Mix-RL 需要 150-180K

### 4.4 分析

#### 4.4.1 Policy-Gradient vs Top-k 蒸馏

两者表现相当（归一化得分 0.937 vs 0.909）。在同源教师下，学生 rollout 集中在教师高概率区域，两种梯度估计器收到相似信号。

#### 4.4.2 同源教师稳定蒸馏

将数学教师换成更强的 Qwen3-235B-A22B 后，数学性能反而下降。同源教师初始 KL 仅 ~0.04，而大模型教师 KL 高达 ~0.19（5 倍），学生收到大量惩罚性梯度信号，top-k 形式甚至在第 18 步发散。这证明同源教师至关重要。

#### 4.4.3 多轮学生-教师进化

一轮 MOPD 后，以 MOPD 学生为起点重新训练领域教师再做第二轮蒸馏，归一化得分从 0.937 提升至 0.986（+0.049），证明 MOPD 可以持续吸收更强教师的能力。

## 5 讨论

MOPD 结构性地解耦了多领域后训练中的串行依赖，将能力生产（Stage-2）与能力集成（Stage-3）分离。三个工作流价值：并行开发、方案级解耦、风险隔离。

## 6 结论

MOPD 在 Qwen3-30B-A3B 上领先最佳基线 5.5 个归一化得分点，在 MiMo-V2-Flash 上匹配或超越教师。同源教师对稳定优化至关重要，支持多轮迭代进化。
